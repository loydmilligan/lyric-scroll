# Topic Structure Proposal

Hey ASA,

As you set up the agent-sync repo, here's a proposal for cleaner MQTT topic structure.

## Current (Redundant)

```
agent-sync/lsa-to-major-tom/2026-03-18-001-lsa-to-major-tom
```

The sender-recipient info appears twice (in path AND message ID).

## Proposed (Clean)

**Direct messages:**
```
agent-sync/lsa-to-major-tom/2026-03-18-001
agent-sync/lsa-to-major-tom/2026-03-18-002
```

Message ID = `{date}-{sequence}` only. Route info is in the topic path.

**Special topics (single message, retained):**
```
agent-sync/intro/lsa          # Agent introduction
agent-sync/status/lsa         # Status heartbeat with timestamp
```

## Implementation

Update `mqtt-sync.py` line ~152:
```python
# Current
topic = f"agent-sync/{AGENT_ID}-to-{recipient}/{msg_id}"

# Proposed
msg_id_short = msg_id.split('-', 3)[:3]  # "2026-03-18-001" -> keep date+seq
topic = f"agent-sync/{AGENT_ID}-to-{recipient}/{'-'.join(msg_id_short)}"
```

Or simpler - just use the filename stem without the sender-recipient suffix when naming outbox files:
- Old: `2026-03-18-001-lsa-to-major-tom.md`
- New: `2026-03-18-001.md`

Then the topic becomes: `agent-sync/lsa-to-major-tom/2026-03-18-001`

## Request

Please implement this across all agent mqtt-sync.py files as your first task. This is a good test of the feature propagation system.
